## 一、新增 / 强化的伪科学营销话术模式

> 目标文件：`skills/pseudoscience-patterns.md`

1. **"CIPA X.0 级" 自定义标准命名陷阱**
   - 识别：厂商引用 CIPA 等国际标准但版本号与官方编号不对应（如把"CIPA DC-X011-2024"包装为"CIPA 7.0/8.0 级"等口语化命名）
   - 核查：到 https://www.cipa.jp/std/std-list_e.html 比对国际标准官方编号
   - 真实情况：标准是分类编号，没有"X.0级"这种纵向递进的版本号体系

2. **"全球首发 + 技术名词"边界条件缺失**
   - 识别：成熟技术包装为"全球首发"，缺少限定语
   - 核查模板：是消费电子首发？是手机端首发？是特定频段/特定芯片实现首发？
   - 真实情况：很多"全球首发"是细分场景的首发

3. **大单代耐久度提升（>2倍）必须第三方实验室认证**
   - 识别：厂商自报耐久度数据单代翻倍以上（如折叠 200,000→500,000 次）
   - 核查：要求看 SGS、Bureau Veritas、TÜV 等第三方实验室报告，未提供则降级为"厂商口径"

4. **品牌借光营销缺技术披露**
   - 识别：以"口袋哈苏""掌上徕卡""军规级"等品牌借光定语作为卖点
   - 核查：要求披露合作的实际技术贡献（色彩科学算法授权？镜头调校？传感器协同？还是仅授权logo）

## 二、新增 / 强化的逻辑核查模式

> 目标文件：`skills/logical-fallacy-catalog.md`

1. **"直接竞争对手同时合作"反常识检测**
   - 识别：报道中出现 A 和 B 是直接竞争对手但同时与 C 合作
   - 例：OpenAI 同时联合高通+联发科开发处理器（两家是手机 SoC 死敌）
   - 核查：要求看到清晰的合作分工或单独签约证据

2. **聚合爆料应拆条核查**
   - 识别：单一报道列出"X 项新特性"或"Y 个升级点"
   - 核查：每一项分别找独立信源，不能把聚合报道作为单一证据使用

## 三、新增 / 强化的事实核查模式

> 目标文件：`skills/fact-check-playbook.md`

1. **TOPS 数字脱离精度无意义**
   - 现有规则强化：当一篇报道出现 TOPS 数字但未声明精度（INT4/INT8/FP16），自动 fail 该条
   - 真实有效的端侧 LLM 指标：模型参数 + 量化精度 + 内存占用 + 首 token 延迟 + 持续 token/s

2. **"整机性能提升 X%" 必须问"对比哪一代/哪个 Benchmark"**
   - 缺基准 → 自动 warn
   - 真实有效的对比：CPU 单核多核 GeekBench / GPU 3DMark / NPU 特定模型 token/s / UFS 顺序读写，分别百分比

3. **单一英文站爆料 + 国内主流媒体未跟进 = 单源风险**
   - 触发规则：在国内 IT之家/微博头部数码博主/快科技 24 小时内未跟进的英文站爆料，credibility 直接降至 0.5 以下
