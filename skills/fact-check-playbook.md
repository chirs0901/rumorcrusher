# 事实核查方法论 / Fact-Check Playbook

> 这是 RumorCrusher 的个人技能库之一。每日质检过程中如果遇到新模式，会自动追加到此文件，逐步进化为越来越锋利的核查武器。

## 一、核查的三层漏斗

### 第一层：参数核对（最快、最硬）
适用于：芯片性能、跑分、相机参数、电池容量、屏幕规格、价格

- **优先源**：官方Newsroom > GSMArena > 厂商发布会PPT > 第三方实测（极客湾、MKBHD）
- **常见陷阱**：
  - "峰值频率"被说成"日常频率"
  - 厂商实验室条件 vs 真实使用条件
  - 安兔兔/GeekBench 不同版本不可直接比较
  - "充电速度" PD/PPS/UFCS/SuperVOOC 协议混淆

### 第二层：交叉验证（中等速度）
适用于：发布会爆料、供应链消息、专利解读

- 最低标准：**至少2个独立信源**报道同一事实
- 注意"循环引用"陷阱：A引用B，B引用C，C引用A，实际只有一个原始源
- 中文社交媒体爆料 → 必须找到英文/官方对应源才能采纳

### 第三层：常识与物理边界（最慢但最关键）
适用于：突破性技术、"颠覆性"声明

- 物理常识：电池能量密度上限、芯片功耗墙、散热极限
- 工程常识：量产爬坡周期、良品率、BOM成本
- 商业常识：定价逻辑、毛利、渠道库存

## 二、高风险标签（触发深度核查）

遇到以下关键词，自动升级为深度核查：

- **绝对化用词**：「全球首款」「行业唯一」「秒杀」「碾压」「断崖式领先」
- **未量化承诺**：「续航大幅提升」「拍照体验飞跃」「AI能力革命性」
- **物理违和**：「续航100小时」「-30°C正常使用」「5分钟充满」+ 无前提条件
- **专家化包装**：「实验室数据」「内部测试」「领先国际X年」
- **情绪驱动**：「震惊」「碾压」「打脸」「翻车」「神机」

## 四、Benchmark-Driven QA 评估框架（自我评估）

> 借鉴 AVeriTeC（学界标杆）+ RealFactBench（LLM 实战 + 不确定性）+ MultiCW（多语言筛查）+ TSVer（时序声明）的方法论组合。每日运行后，对自己的判定进行打分，追踪能力进化曲线。

### 4.1 采用 AVeriTeC 的 4 级官方标签（替代原 pass/warn/fail 三档的简化）

每个 claim 必须用以下四个之一：

| 标签 | 含义 | 触发场景 |
|---|---|---|
| **Supported** | 有充分证据支持 | 至少 2 个独立可信信源（或 1 个官方源 + 1 个第三方）确认；数据可在权威源（GSMArena、官方）查到 |
| **Refuted** | 有证据反驳 | 找到反例、物理违和、原始信源被歪曲 |
| **Not Enough Evidence** | 证据不足 | 单一弱信源、未被验证、无法在 24 小时内交叉确认 |
| **Conflicting Evidence (Cherry-picking)** | 证据冲突或选择性引用 | 不同信源给出矛盾结论；或厂商引用最有利数据而忽略不利数据 |

→ 在 `synthesis.json` 中新增字段 `averitec_label`

### 4.2 借用 RealFactBench 的 Unknown Rate（UnR）—— 允许"我不知道"

**不要强行 pass/fail**。当出现以下任一情况，主动给 `verdict: unknown`：
- 信源单一且无法在 24 小时内交叉验证
- 涉及内部测试数据（如厂商"实验室数据"），但无法获取测试方法
- 时间窗超出（如 2028 年的预测）

**每日跟踪 UnR**：当日 Unknown 数 / 总条数 = UnR
- **健康区间：5%~25%**
- UnR < 5%：可能"过度自信"，对模糊数据强行判断 → 警示
- UnR > 25%：可能"过度保守"，缺少深度核查 → 警示

### 4.3 借用 MultiCW 的"check-worthiness 上游筛查"

**采集后、深度核查前，先对每条素材打 check-worthiness 分（0-1）**：

| 分数 | 处理方式 |
|---|---|
| **≥ 0.7** | 高争议/高影响 → 全流程多 Agent 核查（包括视频源转录） |
| **0.4 ~ 0.7** | 中等 → 标准多 Agent 核查 |
| **0.2 ~ 0.4** | 偏低 → 只跑 fact-check + sentiment 两个 Agent（节省预算） |
| **< 0.2** | 太低（如纯定价公告） → 只标记不深查，作为"背景信息"列在干净报告 |

**check-worthiness 高分的特征**：
- 涉及性能数据/对比/排名
- 涉及"全球首发"、"突破性"、"颠覆"等绝对化用词
- 涉及尚未发布的产品（爆料/泄露）
- 涉及社会争议（涨价、隐私、安全）

### 4.4 借用 TSVer 的时间序列声明核查

凡是声明里出现 **"同比"、"环比"、"提升 X%"、"突破 Y"** 这类数值时序对比，自动触发：
1. 锚定**对比基准时间点**（去年同月？上代旗舰？哪个 Benchmark？）
2. 锚定**测量方法**（GeekBench？AnTuTu？哪个版本？同温度同电量？）
3. 任一锚定无法回填 → 标记为 `Cherry-picking` 或 `Not Enough Evidence`

### 4.5 借用 RealFactBench 的 4 维评分（解释质量）

每个 verdict 不只是标签，还要给一个简短的 **explanation_quality** 自评（1-5）：
- 1：仅一个标签，无理由
- 2：一句话理由
- 3：有理由 + 一个证据链接
- 4：有理由 + 多证据 + 反方证据
- 5：有理由 + 多证据 + 反方证据 + 物理/工程常识对照

→ 每日均值 < 3 视为质量回退，触发方法论补强

### 4.6 每日自评卡（写入 `YYYY-MM-DD/06-self-eval.md`）

每日运行完，输出一份简短自评，6 个指标，方便趋势追踪：

```yaml
date: 2026-05-15
total_items: N
averitec_distribution:
  supported: N1
  refuted: N2
  not_enough_evidence: N3
  conflicting: N4
unknown_rate: 0.XX        # 健康区间 0.05~0.25
avg_explanation_quality: X.X   # 目标 >= 3.5
check_worthiness_high_pct: 0.XX  # 高争议素材占比
time_series_claims_caught: N    # 时序声明触发数
methodology_delta_count: N      # 本日新沉淀方法论数
```

每周写一篇 `wiki/topics/qa-trend-week-N.md` 复盘 7 日 UnR、解释质量、AVeriTeC 标签分布的趋势。

### M-014：多厂"同期"发布声明核查规范
> 新增于 2026-05-29 晚报（触发案例：E-031 骁龙8EG6/天玑9600/苹果A20"同期量产"→Refuted）

**规则**：当声明包含多个SoC或整机厂商"同期"/"同步"推出产品时，必须交叉核查各品牌实际发布窗口（通常以官方发布日为准）。
- **判定标准**：超过4周差距 → 不应使用"同期"，应改为"同季度"或标注时间差
- **高发场景**：苹果/高通/联发科每年Q3-Q4密集发布期，媒体常用"同期"简化叙述
- **处理方式**：保留各品牌实际发布时间，辟谣"同期"说法，全条标记Refuted

### R-2026-05-30-01：汽车电池路线图迁移谬误识别
> 新增于 2026-05-30 晚报（触发案例：三星Galaxy S26全固态电池声称 → Refuted）

**规则**：当声称手机产品将采用"固态电池"时，必须区分汽车动力电池路线图与手机消费电池的不同TRL。
- **判定标准**：声称来源为汽车电池厂商（CATL/松下/三星SDI）路线图，或未指明具体应用场景 → 标记ConflictingEvidence或Refuted
- **验证方法**：检查手机厂商官方新闻稿是否明确声明；查询手机级固态电池TRL（当前约4-5级）
- **手机固态电池商用门槛**：能量密度≥300Wh/L（已达），循环寿命>1000次（部分可达），体积膨胀<10%（未达），BOM成本（未达）→ 当前不具备大规模商用条件

### R-2026-05-30-02：非官方芯片命名标注规则
> 新增于 2026-05-30 晚报（触发案例："骁龙8E6"命名为媒体推测）

**规则**：媒体预测性芯片命名需在条目中标注"非官方"，不得作为事实引用。
- **判定标准**：高通/苹果/联发科/华为官方未正式发布的型号名称，一律标注"[非官方命名]"
- **影响**：含非官方命名的技术参数声称，AVeriTeC降级为ConflictingEvidence

---

### R-2026-05-30-03：存储价格数据原始来源链要求
> 新增于 2026-05-30 晚报（触发案例：DRAM合约价Q1涨80-90%数据缺乏原始来源）

**规则**：凡涉及DRAM/NAND价格涨跌幅的具体百分比，必须追溯至TrendForce/IDC/Gartner原始报告。
- **判定标准**：单季涨幅超40% → 强制标记ConflictingEvidence待核实；原文未链接原始报告 → 降级
- **可靠源**：TrendForce集邦咨询原始报告 > Counterpoint存储季报 > IC Insights

### R-2026-05-30-04：工业传感器≠手机传感器推断
> 新增于 2026-05-30 晚报（触发案例：索尼IMX06A-AJ1R-J被推断"适合旗舰手机"）

**规则**：制造商官方定位为工业/专业传感器的产品，不可推断其将用于消费手机。
- **判定标准**：官方明确标注工业/industrial用途 → ConflictingEvidence；需有OEM采购证据才可升级为Supported
- **MIPI接口兼容性说明**：接口兼容≠商业可行性，需独立评估BOM成本和供货协议
